# 多因子合成中的量纲陷阱：从 Bug 到对照实验，再到诚实的修补

> 探讨日期：2026-05-04  ·  作者：Cyber Quant Arena  ·  分类：Alpha / Multi-Factor Combination

## 1. 提出的问题

在我们这套美股模拟交易系统里，A 组（A01-A10）使用 **Alpha158 风格的多因子组合**进行选股。每天对 Russell 1000 的每只票计算约 30 个因子（KMID、ROC_5/10/20、RSI_14、BBPOS_20、BETA_5...），合成一个 score，排序后买 Top 10。

那么问题来了：**这 30 个因子，量纲完全不一样——RSI_14 范围 [0, 100]，BETA_5 范围 ±1e-5，ROC_5 范围 ±0.01——怎么"加起来"才合理？**

这个看似平凡的工程问题，在量化里叫 **factor combination / composite scoring**，是多因子模型从"有信号"走到"能下单"之间最容易被忽略的一步。

---

## 2. 现状：我们当前是怎么做的

翻开 `factors/signal.py`，第 57 行：

```python
scores[ticker] = float(vals.mean())   # ← 跨因子算术平均
```

逻辑非常直白：**对每只票，把它在 ~30 个因子上的当前值直接做算术平均**，然后所有票的 score 截面排序取 Top N。

> 这就是后文称为 **V0** 的方案。

### 这个方案的隐藏 bug

各因子量纲对比：

| 因子 | 典型量纲 |
|---|---|
| KMID, ROC_5 | ±0.01 |
| MA_RATIO_5 | ~1.0 |
| BBPOS_5 | ±2 |
| **RSI_14** | **0 ~ 100** ⚠️ |
| BETA_5 | ~1e-5 |

`mean(KMID, RSI_14, BETA_5, …)` 的结果几乎完全由 RSI_14 决定——它比其它所有因子大几个数量级。换句话说，**A01-A10 的 composite 模式，本质上是个"伪装的 RSI_14 反转策略"**，其它 29 个因子都是噪声尾数。

更糟糕的是，因子列表里大量同族重复：`ROC_5/10/20`、`MA_RATIO_5/10/20`、`BBPOS_5/10/20`、`BETA_5/10/20`——这些组内相关性极高（实测 ρ ≈ 0.65–0.80），相当于给"动量"这个风格悄悄投了 9 票，给"价值"投了 0 票。

---

## 3. 提出的新方案

最简单的修复：**先把每个因子在截面上做归一化（rank → [0, 1]），再等权求和**。

```python
factor_ranks = cross_section_df.rank(axis=0, pct=True)  # 每个因子独立 rank
composite    = factor_ranks.mean(axis=1)                # 跨因子等权平均
```

这样所有因子量纲被压平到 [0, 1]，每个因子贡献"一票"，RSI_14 不再霸占整个 score。

> 这就是后文的 **V1** 方案。

---

## 4. 业界的标准做法

V1 已经能解决量纲问题，但还不够"工业级"。中信、华泰、Barra 这套卖方/对冲基金标准管线长这样，每一步都在**当日截面**独立完成：

```
原始因子值 → MAD去极值 → 缺失值处理 → 中性化 → z-score → 加权合成 → 排序选股
```

关键步骤：

1. **MAD 去极值**（n=5）：抗厚尾，比 3σ 稳健。
2. **中性化**：对市值、行业（GICS）、Beta 做截面回归取**残差**。剥离风格 beta，留下"纯 alpha"。
3. **z-score 标准化**：保留距离信息（rank 会丢掉）。
4. **ICIR 加权**：`w_i = mean(IC_i) / std(IC_i)`（滚动 60 天）。让稳定的因子拿大权重，噪声因子被自动稀释。

> 这就是后文的 **V2** 方案。

---

## 5. 对照实验：V0 vs V1 vs V2

我们写了 `research/factor_composite_compare.py`，在**完全一致的 Russell 1000 池子、完全一致的 Alpha158 因子集、完全一致的回测引擎**下，只换合成方法。两个时间窗口对比：

### 5.1 长窗：2024-05-01 → 2026-05-01（2 年）

| 方法 | IC mean | ICIR | 年化收益 | 夏普 | 最大回撤 | 十分位价差 |
|---|---|---|---|---|---|---|
| V0 现状 | -0.011 | -0.07 | 4.04% | 0.26 | -20.9% | -1.8 bps |
| V1 rank+等权 | -0.012 | -0.09 | 6.84% | 0.35 | -29.7% | +0.2 bps |
| V2 工业级 | **+0.003** | **+0.029** | -4.36% | -0.23 | -28.0% | -1.9 bps |

![equity_2y](equity_curves_2y.png)
![ic_2y](ic_timeseries_2y.png)

**几个反直觉的发现**：

1. **三种方法 IC 都接近零**。说明 Alpha158 这套 2014 年的 A 股价量因子，在过去 2 年的美股 R1000 里几乎没真 alpha。
2. **V2 IC 最高、累计收益最差**——典型的"中性化代价"。V0/V1 的 6% 收益主要来自未中性化的市值/行业 beta，不是真 alpha；V2 把它剥掉了，真相浮现。
3. **V1 比 V0 累计收益高，但 IC 实际略差**。差距来自尾部押注的偶然性，不是合成方法本身的提升。

### 5.2 短窗：2026-01-01 → 2026-05-01（YTD 4 个月）

| 方法 | IC mean | ICIR | 年化收益 | 夏普 | 最大回撤 |
|---|---|---|---|---|---|
| V0 现状 | 0.0007 | 0.004 | 19.47% | 1.41 | -7.9% |
| **V1 rank+等权** | 0.0017 | 0.010 | **37.23%** | **2.13** | **-7.2%** |
| V2 工业级 | **0.0096** | **0.094** | 3.08% | 0.16 | -14.0% |

![equity_ytd](equity_curves_ytd.png)
![ic_ytd](ic_timeseries_ytd.png)

**结果完全不一样**：

- **V1 夏普 2.13**——但 ICIR 仅 0.01，统计上和噪声没区别。亮眼业绩主要来自押对了"小盘+某行业"风格 beta。
- **V2 ICIR 从 2 年版的 0.029 跳到 0.094**——3 倍提升。这说明 ICIR 加权机制需要 IC 方向稳定的窗口才能起作用，长窗里 IC 频繁翻转把权重打乱了。
- **V1 - V2 的 34% 年化收益差距 ≈ 风格 beta 暴露**。在 2026 YTD 这种风格占优的环境下，不中性化反而赚钱；一旦风格反转，V1 会比 V2 巨亏。

### 5.3 因子相关性诊断

![factor_corr](factor_corr.png)

组内平均 |相关系数|：

- `MA_RATIO_5/10/20` → **0.80** ⚠️
- `BBPOS_5/10/20` → **0.79** ⚠️
- `ROC_5/10/20` → 0.65
- `STD_5/10/20` → 0.64
- `BETA_5/10/20` → 0.51

确认了直觉：动量/反转类因子在不同时间窗口下高度重合，等权合成等于隐性加权。

## 6. 我们的修复

基于实验结果，我们做了**最小改动的 hotfix**——把生产代码 `factors/signal.py` 从 V0 升级到 V1：

```python
# 之前 (V0)：跨因子算术平均，量纲污染
scores[ticker] = float(vals.mean())

# 现在 (V1)：每个因子先做截面 rank，再等权平均
factor_ranks = cs.rank(axis=0, pct=True)
composite    = factor_ranks.mean(axis=1)
```

为什么不直接上 V2？

- V2 在两个时间窗里**累计收益都最差**——因为它揭示了 Alpha158 在 R1000 上没真 alpha 的真相。如果生产投产 V2，A01-A10 的业绩短期会显著下降。
- V1 在两个窗口里**都不输给 V0**，零下行风险。
- V2 pipeline 已写好，未来用作"alpha 真伪检测器"——任何新因子都跑一遍 V2，看中性化后 ICIR 是否还为正。这是甄别真假 alpha 的金标准。

---

## 7. 新的 V1 方案可能还有哪些问题

诚实列一下我们**没有解决**的问题：

### 7.1 因子重合度仍然存在
V1 只解决了量纲问题，没解决**信息重复加权**。`MA_RATIO_5/10/20` ρ=0.8，rank 之后 ρ 仍然 ≈ 0.8。等权求和等于"动量"风格被算了 ~3 票。修法是相关性聚类后组内等权、组间等权（PCA / hierarchical clustering），或干脆删掉冗余窗口。

### 7.2 没有方向定向
不是所有因子"越大越好"。V1 默认所有因子的 rank 都正向加（mean_reversion 模式手动反向）。如果某个因子在当前市场是反向的（例如反转主导期的 ROC_20），V1 会让它和正向因子相互抵消。修法是按滚动 IC 符号自动定向。

### 7.3 没有风格中性化
V1 没剥离市值/行业/Beta 暴露。当前账户业绩的相当大一部分（实验估计 30%+ 年化）来自风格 beta 而不是 alpha。**风格反转时会一次性吐回去**。修法就是 V2。

### 7.4 因子有效性差异未利用
V1 等权对待所有因子。一个 ICIR=0.5 的因子和一个 ICIR=0.01 的因子贡献相同——后者纯粹在稀释信号。修法是 ICIR 加权（V2 已实现）。

### 7.5 静态权重不适应市场状态
V1 的权重永远是 1/N。在动量主导期和反转主导期，"动量"和"反转"两类因子的相对重要性会切换，V1 无法适应。修法是状态相关的 regime-switching 权重。

### 7.6 中性化用的是市值"代理"
V2 实验里的市值用的是"日均成交额"代理（DB 里没存股本数据）。真实 Barra 模型用的是 log(市值)。这是 V2 实验本身的局限性，不是 V1 的问题，但替换 V2 进生产之前必须修。

---

## 8. 一句话总结

**我们以为是合成方法的 bug，实验后发现真正的问题是因子库本身没真 alpha**。V1 hotfix 解决了"账面上量纲污染"这件丢人的事，但 A 组业绩的根本来源是风格 beta，不是选股能力。这个事实不会因为修了 signal.py 而改变——它只会因为我们换更好的因子或者用 V2 揭穿真相而改变。

V2 不是为了投产，而是为了**让我们看清自己的策略到底有没有 alpha**。下一步该做的，不是继续抛光合成方法，而是用 V2 评估 B 组（GP 挖出来的因子）和未来任何新策略的真实 ICIR——只有 ICIR 在中性化后还为正的策略，才值得花资金去跑。
